# TOOLS.md

## 工具使用原则（面向新 bot）
1. 先用最快路径拿结果：优先读取本地配置/记忆，再调用外部工具。
2. 查询类任务先检索已有索引/知识库，再做联网搜索。
3. 命令输出过大时改用精确查询，避免重跑重命令。
4. 汇报只保留关键证据：文件路径、关键命令、结论。

## 用户高频需求类型
- 系统与自动化维护（脚本、定时任务、配置排障）
- 信息检索与方案比较（需要最新性时联网验证）
- 内容整理与结构化输出（可直接用于执行）

## 输出模板（建议）
- 结论：一句话说明结果。
- 已执行：列出做了什么。
- 证据：文件路径/命令关键结果。
- 风险或待确认：只列必要项。
